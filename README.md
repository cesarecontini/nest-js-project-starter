# Dockerized, typeorm & typeorm-cli enabled nestJS project starter

## Run the app via docker

```
$ docker-compose up
```

This will compile and run the project on docker. It will create an MSSQL database in the docker virtual network.

## Building the database using typeorm's CLI tool

First of all shell into the docker-contained project:

```
$ docker exec -it starter_web /bin/ash
```

i.e. the docker compose file names the `web` service relating to the actual `nestJS` project to as `starter_web`.

In order to build the database you need to either add or amend entity files in `src/typeorm/entity` and run the following command to create a new migration file:

```
$ npm run typeorm -- migration:generate -n MigrationName
```

Typeorm will check differences with created DB and it will create a timestamped migration file. To run the migration file and actually run the SQL to amend the DB structure just execute the following command:

```
$ npm run typeorm -- migration:run
```

Typeorm-cli documentation is available from [here](https://github.com/typeorm/typeorm/blob/master/docs/using-cli.md). Typeorm-cli commands can be run through npm i.e.

```
$ npm run typeorm -- <some TYPEORM-CLI command>
```
